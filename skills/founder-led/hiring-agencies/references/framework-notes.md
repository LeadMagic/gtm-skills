# Hiring Agencies — Framework Notes

## Readiness Gate (WbD Operating Model)

Before hiring any GTM agency, confirm:
- [ ] ICP documented (`gtm-context`, `icp-scoring`)
- [ ] Sales process stages defined (`pipeline-management`)
- [ ] Approved messaging exists (`positioning-messaging`, `sales-enablement`)
- [ ] Suppression list maintained
- [ ] CRM configured for agency activity logging

Operating Model score <6 → fix process before outsourcing.

## Agency Types for GTM

| Type | Hire When | Graduate To |
|---|---|---|
| Outbound/SDR | Testing outbound channel, founder bottlenecked | SDR/AE hire |
| RevOps/Clay | CRM messy, enrichment manual, no automation owner | Sales Ops hire |
| Demand gen | Paid/content channel test | Marketing hire |
| Sales consulting | No documented playbook | `pipeline-management` + `sales-enablement` |
| Dev shop | Feature/integration sprint | Engineering hire |

## Pilot Metrics by Agency Type

**Outbound:** qualified meetings/month, cost per meeting, bounce rate, reply rate
**RevOps:** % contacts enriched, CRM field completion, workflow error rate
**Demand gen:** MQL→SQL conversion, CAC by channel, pipeline sourced

## Management Cadence

- Weekly: 30-min scorecard review
- Monthly: pilot checkpoint — continue, adjust, or kill
- Day 90: formal go/no-go decision

## Agent Use

1. Assess Operating Model readiness before recommending agency.
2. Match agency type to stated gap — do not default to outbound.
3. Every pilot needs numeric targets and kill switch.
4. Include in-house graduation path when channel proves out.
