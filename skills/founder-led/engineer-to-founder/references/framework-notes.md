# Engineer to Founder — Framework Notes

Reference tables for `SKILL.md`. Technical founder transition — build, sell, hire.

## Primary frameworks

- **Patrick Collison (Stripe)** — Speed as strategy; ship before incumbents respond.
- **Paul Graham** — Maker's schedule vs manager's schedule; protect deep work blocks.
- **DHH (37signals)** — Profitable from day 1; charge early.
- **Pieter Levels** — Build in public; iterate from day-0 feedback.

## Quit-your-job decision matrix

| Signal | What it means | Action |
|---|---|---|
| Financial safety | 12+ months runway OR revenue covers basics | Can go full-time |
| Traction signal | Paying customers or strong LOIs | Proof to commit |
| Obsession signal | Problem dominates waking hours | Founder-fit signal |
| Market timing | Window closing (regulation, platform shift) | Move now |

## Engineer anti-patterns

| Anti-pattern | Fix |
|---|---|
| 6 months solo coding, no users | 5 user conversations before next line of code |
| Launch when perfect | Ship for ONE user; iterate daily |
| Feature requests = roadmap | Ask "what problem?" not "what feature?" |
| Stack for resume | Fastest stack you ship in |
| Avoiding sales | `founder-sales` from day 0 |
