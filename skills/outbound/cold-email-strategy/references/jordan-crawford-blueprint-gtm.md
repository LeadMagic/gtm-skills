## Core Thesis

Companies buy because they're in **acute pain**, not because they match firmographics. Replace spray-and-pray with **pain-based segments** and messages so valuable prospects would pay to receive them (**PVP**).

> "Would you respond to the messages your AI SDRs are sending?" — Blueprint GTM

**Note:** Jason Bay (Outbound Squad) also uses "PVP" (Permissionless Value Proposition). Crawford's version emphasizes **unique public-data combinations** and AI research depth — cite context when both apply.

## FIND Process (Cannonball GTM)

| Phase | Action | Output |
|---|---|---|
| **F — Focus** | Work backwards from closed deals; define **Pain-Qualified Segment (PQS)** | Segment definition: "who is in the exact situation past buyers were in" |
| **I — Investigate** | Source 2–5 unique public datasets; find **Existential Data Point** for the industry | Enriched list with pain signals |
| **N — Narrate** | Build **Permissionless Value Proposition (PVP)** — insight-led message | Email/LI copy that leads with their pain + data-backed insight |
| **D — Deploy** | Ship campaign; iterate on message quality as atomic unit | Live sequence + reply learning |

**Existential Data Point:** The metric that separates thriving vs failing businesses in a vertical (e.g., utilization rate, compliance deadline, cost per unit).

---

## Pain-Qualified Segments (PQS)

Define segments by **detectable acute pain**, not title + employee count:

1. Interview 5–10 closed-won customers — why did they buy **now**?
2. Extract situational triggers (regulatory change, stack migration, hiring spike)
3. Build list using signals that indicate that situation **today**
4. Validate: would this person recognize the problem in touch 1 without explanation?

---

## Permissionless Value Proposition (PVP)

A message must pass the **pay-to-receive test**:

- [ ] Uses data the prospect doesn't have but needs
- [ ] Combines 2+ public sources they wouldn't merge themselves
- [ ] Leads with **their** pain/opportunity — zero product pitch in line 1
- [ ] Actionable without a meeting (insight stands alone)

**Evaluate any AI SDR / outbound tool:** read the actual message. Would you reply?

## Cannonball GTM — Four Phases (Strategic)

1. **Focus** — Map pain-based segments; identify existential data point
2. **Investigate** — Aggregate unique public data sources
3. **Narrate** — Craft PVPs per segment
4. **Deploy** — Agentic GTM: AI for research + structure; human for strategy

Cannonball live builds: full agentic GTM for a random company in ~1 hour (public demos).

| Message draft | Prompt: PQS context + PVP rules → iterate on atomic message |
| Scale | Don't scale bad messages — fix PVP before adding volume |

## When to Use

- Stuck at 1–3% reply rates despite good deliverability
- List building from **won deals**, not TAM spreadsheets
- Evaluating AI SDR vendors (message-quality test)
- Creative outbound for narrow verticals with public pain signals
- `list-building` + `cold-email-copywriting` research layer

## When NOT to Use

- No closed deals to work backwards from — run founder sales first
- Brand-new domain with no warmup — fix infra before creative copy
- Post-discovery deal coaching — use Keenan Gap Selling / MEDDICC

---

## PVP Quality Checklist

- [ ] Segment defined by pain situation (PQS), not just firmographics
- [ ] Message uses 2+ public data points merged uniquely
- [ ] First line is about **them**, not you
- [ ] Prospect could act on the insight without your product
- [ ] You would personally reply if you received it
- [ ] Product mention deferred until conversation, not touch 1

---

## Anti-Patterns

| Mistake | Fix |
|---|---|
| Demographic ICP only | PQS from won-deal analysis |
| AI volume without message QA | One great PVP → then scale |
| Generic "personalization" (name + company) | Data-combination insight |
| Scaling AI SDR before PVP works | Manual 50 sends; iterate message |
| Copying Jason Bay PVP without research depth | Bay = messaging relevance; Crawford = data moat |
| Enrichment stack without copy structure | Add Pat Spielmann Hook-Line-Sinker — `../cold-email-copywriting/references/pat-spielmann-outbound-copy.md` |
