## Core Philosophy

Cold leads become warm when the prospect believes **your solution fits their problem** — not when you explain features. Problem-first, always.

Multichannel + personal brand (LinkedIn content via Taplio) increases reply rates because trust exists before the ask.

---

## Optimal Cold Email Structure

| Section | Purpose |
|---|---|
| **Subject line** | Short, lowercase, 2–3 words — mimics internal mail |
| **Introduction** | Personalized icebreaker (1–3 sentences) — specific observation |
| **Situation / Problem** | Name the problem + root causes in their context |
| **Value / Solution** | Outcome framing — not feature list |
| **Social proof** | Credible, non-BS proof (logo, metric, micro-case) |
| **CTA / CTC** | **Call to Conversation** — validate interest, not hard close |

**CTC example:** "Worth exploring if [problem] is on your radar this quarter?"

## Sequence Architecture

| Rule | Detail |
|---|---|
| **Touch count** | 4–9 follow-ups for cold outreach |
| **Timing** | ≥2 days between touch 1–2; widen gaps on later touches |
| **Multichannel** | Email + LinkedIn + phone in one coordinated flow (lemlist native) |
| **Follow-up content** | Each touch adds **new value** — never "just following up" |
| **Benchmarks** | lemlist published 17–30% reply rates on optimized multichannel sequences |

---

## Trigger → Problem Mapping

1. List problems your ICP faces
2. For each problem, define **external triggers** you can detect (hiring, funding, tech stack, headcount)
3. Keep only triggers you can **actually source** (Sales Nav, lemlist DB, web scrape, AI enrichment)
4. Open with trigger → problem → CTC

---

## Decision Hill (MR Framework)

Prospects decide on a "hill" — momentum matters. Sequence design should:

- Front-load relevance (problem recognition)
- Escalate channel mix if email stalls (LinkedIn connect → DM → call)
- Use lemwarm + mailbox rotation for deliverability (see `lemlist-setup`, `email-deliverability`)

## Multichannel Stack (lemlist-native)

| Channel | Role in sequence |
|---|---|
| **Email** | Primary narrative; problem + proof |
| **LinkedIn** | Connection, voice note, DM — different copy, same problem thread |
| **Phone** | Post-engagement or high-ACV accounts |
| **Enrichment** | Waterfall providers inside lemlist; AI variables (Perplexity) for T3 |

## When to Use

- Multichannel sequence design in lemlist (or equivalent SEP)
- Problem-first copy structure (pairs with Becc Holland pillars, Josh Braun sorting)
- Bootstrap outbound GTM — high reply rate without large SDR bench
- Personal brand + outbound combo (Taplio/LinkedIn → lemlist)

## When NOT to Use

- Pain-qualified segment **research** at depth — add Jordan Crawford PQS/PVP
- Enrichment + verify-before-send + Hook-Line-Sinker copy — Pat Spielmann `../cold-email-copywriting/references/pat-spielmann-outbound-copy.md`
- Enterprise qualification / discovery depth — add Keenan Gap Selling
- Domain/DNS setup — use `domain-infrastructure` first

## Campaign Checklist

- [ ] Subject: ≤3 words, lowercase, no spam triggers
- [ ] Touch 1 is problem + situation, not product pitch
- [ ] Social proof is specific (named customer or metric)
- [ ] CTC is conversational question, not "15 min Tuesday?"
- [ ] 4–9 touches planned with unique angles per step
- [ ] LinkedIn steps written for LinkedIn — not email copy-paste
- [ ] lemwarm active; daily send limits respected
- [ ] Triggers mapped to problems you can actually detect

---

## Anti-Patterns

| Mistake | Fix |
|---|---|
| Feature-first email | Situation/problem before solution |
| "Just checking in" follow-ups | New insight, case study, or trigger per touch |
| Same copy on email + LinkedIn | Channel-native rewrite |
| Ignoring deliverability | lemwarm + rotation before scaling volume |
| No personal brand | LinkedIn presence improves outbound conversion (lempire model) |
