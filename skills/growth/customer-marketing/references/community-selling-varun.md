## Community selling vs customer marketing

| Customer marketing (`customer-marketing` SKILL) | Community selling (this doc) |
|---|---|
| References, case studies, NPS advocates | **Prospects and users** learning from peers before sales |
| Post-sale advocacy ladder | Pre-sale **trust transfer** from community credibility |
| G2 reviews, reference calls | Slack/WhatsApp/MSP groups, Clay Clubs, creator content |

Both compound — community creates champions; advocacy program formalizes them.

## Clay playbook (public, Varun Anand)

### Phase 1 — Infiltrate existing communities (don't build yet)

Varun joined communities where **agency owners and GTM builders** already gathered:

- [Modern Sales Pros](https://www.moderndatalab.com/) (search history for "data enrichment", "outbound")
- Private Slack groups (e.g. SaaS Yacht Club, Sales Technicians — Eric Nowoslawski)
- WhatsApp groups for agency owners
- Keyword alerts (Slack native + tools like Syften) for problem phrases Clay solves

**Rule:** Respond to **problems**, not product pitches. Value-first DMs when someone describes a relevant pain.

### Phase 2 — Reverse demo → Slack capture

Every sales call ended with:

1. Force join **public Slack** (Clay removed Intercom — support only in community)
2. Prospect DM when joined — support becomes peer-visible
3. Questions in Slack → **content videos** for reuse

Cross-link: `demo-scripts` → `reverse-demo-varun.md`

### Phase 3 — Own the ecosystem layer

| Layer | Purpose |
|---|---|
| **Slack community** | Support, peer learning, product feedback (15k–20k+ members public estimates) |
| **Clay Clubs** | 60+ city chapters — IRL events, table sharing |
| **Clay University** | Education before sales — async + cohorts |
| **Creators / Experts** | Agency owners post workflows on LinkedIn — organic distribution |
| **GTM Engineer** title | Category creation — job market pulls product into orgs |

**Principle (Databox / Clay ecosystem):** Start with **education**, not sales. Certifications and university precede talent marketplace and enterprise sales.

## Community selling operating model

```
Listen (keyword alerts) → Help (DM/thread) → Demo (reverse) → Onboard (Slack) → Champion (creator/cert)
```

| Step | Owner | KPI |
|---|---|---|
| Social listening | GTM / growth | Time-to-reply in target communities |
| Helpful reply → call | AE or founder | Calls booked from community (not cold) |
| Reverse demo | AE | Demo → paid same week |
| Slack join rate | AE | % demos joining community same call |
| Peer answers | Community team | % tickets answered by members <24h |
| Creator posts | Ecosystem | Workflow posts / month on LinkedIn |

## Slack-first support decision

Clay's radical move: **remove Intercom**, mandate Slack for help.

| Benefit | Risk |
|---|---|
| Support scales via peers | Noisy for enterprise buyers expecting private support |
| Every question = content seed | Requires moderation headcount at scale |
| Prospects see active community | Bad UX questions are public |

**When to copy:** PLG/hybrid with technical users who tolerate public Slack. **When not to:** Enterprise-only with strict confidentiality — use private Slack Connect or customer portal instead.

## Launch checklist (mid-market SaaS)

- [ ] Identify 3–5 **existing** communities (not build new first)
- [ ] Set keyword alerts for problem language
- [ ] Assign owner for **<1 hour** community response SLA
- [ ] Bridge from community → **reverse demo** booking link
- [ ] End every demo with **single community CTA** (join link + DM confirm)
- [ ] Track `source = community` in CRM
- [ ] Month 3+: consider single branded Slack + monthly office hours
- [ ] Month 6+: certify 10 champions (references + creator program)

---

## Anti-patterns

1. **Spammy seeding** — kicked from MSP/Slack groups fast; add value first
2. **Community without product depth** — churn after hype cycle
3. **Sales reps-only community** — Clay won with **users who teach users**
4. **Intercom + Slack** — split support; Clay chose one channel deliberately
