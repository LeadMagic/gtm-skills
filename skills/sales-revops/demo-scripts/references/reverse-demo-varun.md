## What it is

Traditional demo: rep shares screen → feature tour → prospect watches.  
**Reverse demo:** prospect shares screen → solves **their real problem live** → rep guides with annotations and questions.

Clay's goal after 100+ sessions: **"a-ha" moment in minutes**, most converting on the same call.

---

## When reverse demo fits

| Fit | Poor fit |
|---|---|
| Complex product requiring hands-on learning | Simple UI — 5-minute self-serve suffices |
| Prospect has a specific workflow pain | No problem articulated — need discovery first |
| Trial or freemium signup is frictionless | Heavy security review before product access |
| Rep can guide via Zoom/Meet annotations | Product requires SE for every session |
| PLG + sales hybrid | Pure enterprise RFP — may still use for champion enablement |

---

## Pre-call checklist (rep)

- [ ] Email prospect: come with **one concrete use case** (data problem, workflow, list)
- [ ] If no use case: reserve first **5 minutes** to co-design one (don't wing 30 min)
- [ ] Trial/signup link ready — prospect creates account on their machine
- [ ] Zoom annotation / pointer enabled
- [ ] **Slack community invite** ready (Clay: support continues in public Slack)
- [ ] Internal note: treat session as **product feedback** — log UX friction live

---

## Call structure (30 minutes)

| Minute | Who leads | Action |
|---|---|---|
| 0–3 | Rep | Recap use case; confirm success = "you solve X by end of call" |
| 3–8 | Rep + prospect | If needed: define workflow steps on shared doc |
| 8–10 | **Prospect** | Prospect shares screen; signs into product |
| 10–25 | **Prospect** (rep guides) | Prospect clicks; rep annotates — **2:1 control ratio** (prospect controls 2× rep talk time) |
| 25–28 | Prospect | Prospect completes "first win" — one output they keep |
| 28–30 | Rep | Next step + **join community** + schedule only if needed |

**Rule:** Do not take screen back for a feature tour unless prospect is stuck >3 minutes on one step.

---

## Talk tracks

**Opening (rep):**
> "Today isn't a presentation — you're going to solve [problem] live. I'll guide, but you're driving. Fair?"

**Handoff (rep):**
> "Share your screen, open [signup link], and I'll point where to click. Goal: [specific output] before we hang up."

**When prospect stalls (rep):**
> "What would you try next?" → wait → annotate only if stuck 90+ seconds

**Close (rep):**
> "You just built [artifact]. Join [Slack/community] — that's where we answer follow-ups. Post there before booking another call."

## First Win strategy

Design one **task-based path**, not a feature menu:

| Bad path | Good path |
|---|---|
| "Let me show dashboards, integrations, API…" | "Enrich 50 accounts from your ICP and export to CSV" |
| 15 features in 30 min | 1 workflow, 3–5 steps, tangible output |

Clay pattern: solve stated enrichment/outbound problem in **≤30 minutes**; blow mind with **quality of data or speed**, not breadth.

## Conversion & product loop

Varun's public outcomes from Clay's reverse demo era:

- Faster time-to-"a-ha" (multi-call → single call)
- Higher conversion when prospect **owns** the workflow
- **Product improvement loop:** ideal customers expose bugs and UX gaps in real time — tighter than survey feedback

**RevOps tracking:**

| Metric | Baseline vs reverse |
|---|---|
| Demo → opp | Track separately by demo type |
| Demo → closed won | 30/60/90 day |
| Time on call where prospect has screen control | Target >60% |
| Support tickets week 1 post-demo | Should fall vs tour demos |

## Integration with traditional demo skills

| Traditional (`demo-scripts` templates) | Reverse demo overlay |
|---|---|
| Discovery recap opens demo | Use case **defined before** screen share |
| Feature → pain → outcome | Step → prospect click → outcome |
| Differentiator moment | "First win" moment prospect builds themselves |
| Next steps | Community join + async support before second meeting |

**Competitive displacement:** Pair with Great Demo! **client-led incumbent demo** in discovery (`meeting-prep`), then reverse demo **your** product in evaluation.

## Certification (enablement)

Before reps run reverse demos solo:

1. Shadow 3 reverse demos
2. Run 2 with manager scoring **prospect control %** and **first-win achieved Y/N**
3. Log 5 product feedback items from sessions — proves feedback loop is live

See `revenue-team-onboarding` certification rubric — add "Reverse demo" module for PLG-hybrid products.
