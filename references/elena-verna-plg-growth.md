## Core Model — Growth Loops Over Funnels

PLG companies grow through **loops** (user action → output → attracts more users), not linear funnels alone.

| Loop type | Mechanism | B2B SaaS example |
|---|---|---|
| **Viral** | User invites others | Collaboration invites, shared workspaces |
| **Content** | User output is discoverable | Public dashboards, templates gallery |
| **Paid** | Revenue funds acquisition | Sales-assist on PQLs |
| **Sales-assist** | Product usage triggers human touch | PQL → AE when account hits threshold |

**Rule:** Map your primary loop before scaling paid — otherwise CAC payback breaks.

---

## PQL (Product Qualified Lead) Framework

1. **Define activation event** — action correlating with retention (not signup)
2. **Score accounts** — usage depth + firmographics + intent
3. **Route tiers** — self-serve vs sales-assist vs enterprise
4. **Measure TTV** — time from signup to activation event

Pair with `freemium-optimization`, `tracking-plan`, `plg-strategy`.

## PLG + Sales Hybrid

Verna advocates **both** — not PLG *or* sales:

- PLG for acquisition and activation
- Sales for expansion, enterprise, and high-ACV segments
- **Same data model** — product events feed CRM (see Amplitude → Salesforce patterns)

**Contrast Ali Ghodsi:** Databricks = open-source → enterprise sales. Verna = SaaS PLG → sales-assist overlay.
