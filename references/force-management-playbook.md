# Force Management Playbook

**Sources:** [Force Management](https://www.forcemanagement.com/) · [Command of the Message](https://www.forcemanagement.com/command-of-the-message) · [MEDDICC](https://www.forcemanagement.com/meddicc) · [John Holland](https://www.linkedin.com/in/johnholland/) (CEO) · Andy Whyte — [MEDDICC.com](https://meddicc.com/) (operationalizes FM MEDDICC)

Force Management and Winning by Design overlap on recurring revenue but diverge on **sales enablement**, **territory/pod design**, and **cross-functional alignment cadence**. Use WbD for SPICED/Bowtie/GTM Index; use Force Management for Command of the Message, MEDDICC qualification rigor, and revenue-org alignment at scale.

## 1. Cross-Functional Alignment Cadence

| Rhythm | Participants | Purpose | Outputs |
|---|---|---|---|
| **Weekly** | Frontline AE/SDR + manager | Pipeline hygiene, MEDDICC gaps, next-step accountability | Updated CRM, 3–5 deal actions per rep |
| **Weekly** | SDR + AE + marketing ops | Lead quality, SLA, handoff defects | MQL→SQL conversion notes, ICP feedback |
| **Bi-weekly** | AE + SE + CSM (active deals) | Technical fit, implementation risk, expansion signals | Mutual action plan updates |
| **Monthly** | Sales + Marketing + CS leadership | Funnel conversion, campaign ROI, churn drivers | One-page GTM scorecard |
| **Monthly** | Enablement + sales managers | Playbook gaps, win/loss themes, certification | Updated battlecards / talk tracks |
| **Quarterly** | CRO + CMO + CCO | Territory design, quota model, pod economics | Headcount plan, quota letters |
| **Quarterly** | Board / exec team | Unit economics, CAC payback, attainment distribution | ARR bridge, hiring triggers |

**Alignment rule:** Marketing owns *demand quality* (ICP-fit MQLs). Sales owns *conversion velocity* (stage duration, win rate). CS owns *retention economics* (NRR, expansion). RevOps owns *single source of truth* (definitions, fields, dashboards).

## 2. Reporting Structures by ARR Stage

### ~$1M ARR — Founder-Led / First Team

| Layer | Span of Control | Roles |
|---|---|---|
| CEO / Founder | 1–2 ICs | Founder sells + 1 full-stack AE |
| — | — | Optional: 1 SDR when AE capacity maxed |

**Manager ratio:** None yet. Founder = player-coach.  
**Quota:** Founder + AE each carry full quota; no overlay SE unless deal size >$50K ACV.

### ~$10M ARR — Repeatable Motion

| Layer | Span of Control | Roles |
|---|---|---|
| VP Sales / CRO | 2–3 managers | Player-coach manager per pod |
| Sales Manager | 6–8 ICs max | 4–6 AEs + 2–4 SDRs per pod |
| SE (overlay) | 1 SE : 3–4 AEs | Complex deals only |
| CS Manager | 1 : 15–25 accounts | CSMs by segment (SMB/MM) |

**Manager ratio:** 1 manager : 6–8 ICs (Force Management / Bridge Group norm).  
**RevOps:** 1 FTE per ~$8–12M ARR (stack + reporting).

### ~$50M ARR — Multi-Pod / Multi-Segment

| Layer | Span of Control | Roles |
|---|---|---|
| CRO | 3–5 directors | New business, expansion, SDR org |
| Director | 2–3 managers | Segment or geo pods |
| Manager | 6–8 ICs | AE + SDR pod |
| Enablement | 1 : 40–60 sellers | Certification, Command of the Message upkeep |
| Deal Desk | Shared service | Pricing, legal, MEDDICC audit on >$X deals |

**Span rule:** Never exceed 8 direct reports per manager — coaching quality collapses (Dorsey / FM enablement data).

## 3. Pod Model — Composition & Economics

### Standard Pod Compositions

| Motion | SDR | AE | SE | CSM | Notes |
|---|---|---|---|---|---|
| **Simple** (PLG + sales assist) | 2 | 3 | 0.5 | 1 | High velocity, low touch |
| **Moderate** (mid-market SaaS) | 1 | 2 | 0.5 | 1 | Default B2B SaaS |
| **Complex** (enterprise) | 1 | 1 | 1 | 1 | Long cycle, multi-thread |

**Early-stage evolution (WbD + FM):** 2:1:1 → 1:2:1 → 1:1:1 as AE-led closing matures.

### Pod Economics Worksheet

| Line Item | Formula | Example (Moderate Pod) |
|---|---|---|
| **Pod headcount cost** | Sum(OTE × 1.15 burden) | SDR $70K + 2×AE $300K + SE $140K + CSM $110K = **$620K** loaded |
| **Pod quota (annual)** | AE quota × AE count | 2 × $800K = **$1.6M** new ARR |
| **Pod ARR at 80% attainment** | Quota × attainment | **$1.28M** |
| **Pod cost % of ARR** | Headcount / ARR generated | $620K / $1.28M = **48%** (pre-ramp) |
| **Target pod cost %** | ≤35% at steady-state 80% ramp | Hire/restructure if >40% for 2 quarters |
| **Quota:OTE ratio** | Quota / AE OTE | $800K / $150K = **5.3:1** (Bridge Group norm ~5:1) |
| **SDR meetings → pipeline** | Qualified meetings × opp rate × ACV | 40/mo × 25% × $25K = **$250K/mo pipeline** |

**Decision triggers:**
- Pod cost >40% of ARR for 2 quarters → reduce headcount or raise quota/ACV
- SDR:AE ratio wrong → pipeline starvation (too few SDRs) or junk pipeline (too many)
- SE bottleneck → deals stalling in technical validation; add 0.5 SE or certify AEs

## 4. Unit Economics ↔ Quota ↔ Headcount

Link pod economics to company-level metrics (David Skok + FM operating rhythm):

| Metric | Formula | Stage Benchmark |
|---|---|---|
| **CAC Payback** | CAC / (ARPA × Gross Margin) | SMB <12 mo · MM <18 mo · ENT <24 mo |
| **LTV:CAC** | LTV / CAC | ≥3:1 healthy · ≥5:1 efficient |
| **Magic Number** | Net New ARR (Q) / S&M Spend (Q-1) | >0.75 efficient · >1.0 aggressive |
| **Rep productivity curve** | ARR closed / rep / month by tenure | Month 1–3: 0% · 4–6: 40% · 7–12: 80% · 13+: 100% |
| **Quota attainment distribution** | % reps ≥100% quota | Healthy: 50–60% at quota; <40% = plan problem |

### Headcount Planning Sequence

1. **Math model** — Target ARR ÷ quota per AE ÷ attainment % = AE count
2. **Pipeline model** — Opportunities needed ÷ opp/rep ÷ SDR feed rate = SDR count
3. **Retention model** — Accounts per CSM ÷ NRR target = CSM count
4. **Pod wrap** — Group into pods; run pod economics worksheet
5. **Quarterly true-up** — Attainment distribution + CAC payback → hire/pause/fire
