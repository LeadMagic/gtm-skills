# Support

Use GitHub Issues for public support:

- Skill request: request a new GTM skill.
- Improvement: propose a better framework, trigger, artifact, or output format.
- Bug report: report validator, installer, docs, or marketplace issues.

For private security or sensitive-content issues, do not open a public issue. See `SECURITY.md`.

Before opening an issue:

```bash
npm run check
gh skill publish --dry-run
```

Include the command output when reporting a reproducible problem.
